Guardrail Documentation

Complete API reference and guides for Guardrail, the open-source security and rate limiting toolkit.

Welcome

Guardrail is a comprehensive security toolkit that provides rate limiting, bot detection, attack protection, and IP intelligence for Node.js applications. It runs entirely on your infrastructure with no external dependencies or usage-based pricing.

Key Features

  • Zero-Config — Get started with built-in presets and auto-discovery
  • Rate Limiting — Token bucket and sliding window algorithms
  • Bot Detection — Identify and block automated traffic
  • IP Intelligence — Geolocation with VPN/proxy detection
  • Email Validation — Block disposable and invalid emails
  • Attack Protection — Shield against SQL injection, XSS, and more
  • Framework Support — Next.js, Express, Nest.js, Fastify, and Koa

Documentation Structure

The documentation is organized into the following sections:

Core API

Main methods and functions for creating and using Guardrail instances:

  • guardrail() — Create a Guardrail instance
  • protect() — Evaluate a request against rules
  • checkHealth() — Check system health
  • getStorage() — Get storage adapter
  • getIPService() — Get IP service
  • getMetrics() — Get metrics collector
  • on() — Register event handlers

Rules

Security and rate limiting rules you can apply:

  • shield() — Attack protection
  • bot() — Bot detection
  • window() — Sliding window rate limiting
  • bucket() — Token bucket rate limiting
  • email() — Email validation
  • filter() — Expression-based filtering

Framework Adapters

Framework-specific integrations:
